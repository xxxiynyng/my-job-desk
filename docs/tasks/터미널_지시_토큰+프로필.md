## 작업 2 — 타이포그래피 토큰 마이그레이션

배경: 코드에 손으로 박힌 폰트 크기 `text-[Npx]`가 약 313곳. 디자인 시스템 SSOT 3장이 확정한 위계에 **이름표(토큰)를 붙여 단일 출처화**한다. 크기는 안 바꾸므로 시각 변화 0(예외: 에디터 제목 16→15px 통일 1건).

### 2-0. 시작 전
라이브 레포에서 실제 횟수 재확인: `grep -rhoE "text-\[[0-9]+px\]" src | sort | uniq -c | sort -rn`.

### 2-1. 토큰 정의 (커밋 1)
`tailwind.config.ts`의 `theme.extend`에 추가:
```ts
fontSize: {
  micro:   ['9px',  { lineHeight: '1'   }],
  mini:    ['10px', { lineHeight: '1.2' }],
  chip:    ['11px', { lineHeight: '1.3' }],
  body:    ['13px', { lineHeight: '1.5' }],
  title:   ['15px', { lineHeight: '1.3' }],
  heading: ['26px', { lineHeight: '1.1' }],
},
```
기존 `text-xs`(12)/`text-sm`(14)/`text-base`(16)은 유지된다. 커밋: `chore(ds): 타이포 토큰 6종 정의`.

### 2-2. 기존 토큰으로 치환 — 시각 변화 0 (커밋 2)
- `text-[12px]` → `text-xs`
- `text-[14px]` → `text-sm`
커밋: `refactor(ds): text-[12px]→text-xs, text-[14px]→text-sm`.

### 2-3. 신규 토큰으로 치환 — 시각 변화 0 (커밋 3)
- `text-[11px]` → `text-chip`
- `text-[13px]` → `text-body`
- `text-[10px]` → `text-mini`
- `text-[9px]`  → `text-micro`
- `text-[26px]` → `text-heading`
커밋: `refactor(ds): 칩·본문·카운트·마이크로·H1 토큰 치환`.

### 2-4. 제목 통일 — 유일한 시각 변화 (커밋 4, 단독)
- `text-[15px]` → `text-title`
- `text-[16px]` → `text-title` (DetailEditor.tsx, 16→15px로 통일)
커밋: `refactor(ds): 제목 text-title 통일(16→15)`. **DetailEditor 제목 before/after 스크린샷 확인.**

### 2-5. 헤더 배경 (커밋 5)
