# 랜딩 설문 → 구글 시트 연결 핸드오프 (프론트팀용)

> 작성: 2026-07-22 · 대상: pickd.ai.kr(실도메인) 운영 프론트팀
> 목적: 랜딩페이지 베타 설문 응답을 팀 공용 구글 계정의 스프레드시트에 수집

## 배경 (현재 상태)

- 실도메인 `pickd.ai.kr`은 프론트팀 Vercel에 연결되어 있음 (이 레포의 `pickd-seven.vercel.app`은 프로토타입)
- 프로토타입 레포의 `public/landingpage.html`(커밋 `e2f2288`)에는 이미 제출 로직이 구현됨 — `CONFIG.formEndpoint`로 응답을 POST
- 수신처는 **Google Apps Script 웹 앱**(무료·무제한, 시트에 바로 쌓임). 백엔드 불필요
- ⚠️ 현재 프로토타입에 연결된 엔드포인트는 개인 계정 배포라, **팀 공용 계정(pickd.ai.kr@gmail.com)으로 새로 배포**해서 데이터가 팀 계정에 남게 하는 것이 이 문서의 목적

## 1단계 — 팀 계정으로 시트 + Apps Script 만들기

팀 공용 계정(pickd.ai.kr@gmail.com)으로 로그인한 상태에서:

1. [sheets.new](https://sheets.new) → 시트 이름 "Pickd 베타 설문 응답"
2. 1행 헤더: `제출시각` | `연락처` | `응답(JSON)`
3. 시트 메뉴 **확장 프로그램 > Apps Script** → 기본 코드 전부 지우고 아래 붙여넣기:

```javascript
function doPost(e) {
  const sheet = SpreadsheetApp.getActiveSpreadsheet().getActiveSheet();
  let d = {};
  try { d = JSON.parse(e.postData.contents); } catch (err) {}
  sheet.appendRow([new Date(), d.email || d.contact || '', JSON.stringify(d)]);
  return ContentService.createTextOutput('ok');
}

function doGet() {
  return ContentService.createTextOutput('Pickd survey endpoint OK');
}
```

⚠️ **두 함수가 최상위여야 함** — 다른 함수 안에 중첩하면 웹 앱이 찾지 못함 (실제 발생했던 실수).

## 2단계 — 웹 앱 배포

1. 우측 상단 **배포 > 새 배포** → 유형 **웹 앱**
2. 설정 (둘 다 필수):
   - 실행 계정: **나**
   - 액세스 권한: **모든 사용자** ← 아니면 랜딩에서 전송 차단됨
3. 배포 → 권한 승인("확인되지 않은 앱" 경고 시 고급 > 이동)
4. 발급된 `https://script.google.com/macros/s/…/exec` URL 복사

**운영 주의 2가지**:
- 코드 수정 후엔 **배포 관리 > 연필 > 버전 "새 버전" > 배포**로 반영 (저장만으론 라이브 미반영)
- "새 배포"를 만들면 **URL이 바뀜** → 프론트 코드의 URL도 같이 갱신해야 함. 기존 배포를 "새 버전"으로 수정하면 URL 유지

## 3단계 — 프론트 코드 연결

랜딩페이지의 제출 함수에서 (프로토타입 레포 `public/landingpage.html` 참고, 커밋 `d58ba5d`·`e2f2288`):

```javascript
const FORM_ENDPOINT = '발급받은 /exec URL';

await fetch(FORM_ENDPOINT, {
  method: 'POST',
  // ⚠️ 반드시 text/plain — Apps Script는 preflight(OPTIONS) 미지원이라
  // application/json으로 보내면 브라우저 CORS에 차단됨. 수신부는 동일하게 JSON.parse 함
  headers: { 'Content-Type': 'text/plain;charset=utf-8' },
  body: JSON.stringify(payload),  // payload = 설문 응답 객체 (email 또는 contact 필드 포함 권장)
});
```

프로토타입과 동일 구조를 쓰면 `CONFIG.formEndpoint` 값만 교체하면 됨.

## 4단계 — 검증 체크리스트 (순서대로)

1. [ ] 브라우저에서 `/exec` URL 열기 → "Pickd survey endpoint OK" 표시 (안 되면 2단계 액세스 권한 재확인)
2. [ ] 배포된 랜딩에서 설문 실제 제출 → DevTools Network에 `exec` 요청 발생 확인
3. [ ] Apps Script 좌측 ≡ **실행** 메뉴에 `doPost` "완료됨" 기록 확인 (없으면 요청이 도달 안 한 것 = 프론트 URL/캐시 문제, "실패"면 스크립트 오류)
4. [ ] 시트에 행 추가 확인 → 테스트 행 삭제

## 참고

- 계정 비밀번호는 이 문서에 기재하지 않음 — 팀 내 안전한 채널로 공유할 것
- 수집 데이터에 연락처(이메일/전화)가 포함되므로 시트 공유 범위는 팀 내로 제한 권장
- 향후 백엔드(Spring Boot) 도입 시 이 시트는 그대로 두고 엔드포인트만 교체하면 됨
