# [요청] 랜딩페이지 베타 설문 응답 수집 연결

> 작성: 2026-07-22 (기획 → 프론트팀) · 예상 소요 30분 이내

## 배경

현재 `pickd.ai.kr` 랜딩의 설문·베타 신청 제출이 **저장되지 않고 유실 중**이에요. 8월 베타 모집의 시장 검증 데이터라 우선 처리가 필요해요. 방식은 프로토타입에서 이미 검증한 **Google Apps Script 웹 앱 → 구글 시트** 구성(무료·무제한, 백엔드 불필요)이에요.

## 할 일 1 — 팀 계정으로 수집 시트 만들기

1. 팀 공용 계정 **pickd.ai.kr@gmail.com** 으로 구글 로그인 (접근 정보는 별도 채널로 전달)
2. 새 스프레드시트 생성, 이름 "Pickd 베타 설문 응답"
3. 1행에 헤더 입력: `제출시각` / `연락처` / `응답(JSON)`

## 할 일 2 — Apps Script 웹 앱 만들기

1. 시트 메뉴 **확장 프로그램 > Apps Script** → 기본 코드 전부 삭제 후 아래 붙여넣기
   (⚠️ 두 함수가 **최상위**에 있어야 해요 — 다른 함수 안에 중첩하면 동작하지 않아요. 실제 발생했던 실수)

```javascript
function doPost(e) {
  const sheet = SpreadsheetApp.getActiveSpreadsheet().getActiveSheet();
  let d = {};
  try { d = JSON.parse(e.postData.contents); } catch (err) {}
  sheet.appendRow([new Date(), d.email || d.contact || '', JSON.stringify(d)]);
  return ContentService.createTextOutput('ok');
}

function doGet() {
  return ContentService.createTextOutput('Pickd survey endpoint OK');
}
```

2. 저장 → **배포 > 새 배포** → 유형 **웹 앱**
3. 설정 두 가지 (둘 다 필수):
   - 실행 계정: **나**
   - 액세스 권한: **모든 사용자** ← 아니면 랜딩에서 전송이 차단돼요
4. 배포 → 권한 승인 ("확인되지 않은 앱" 경고가 뜨면 고급 > 이동)
5. 발급된 **웹 앱 URL** (`https://script.google.com/macros/s/…/exec`) 복사

## 할 일 3 — 랜딩 코드에 연결

랜딩페이지 설문 제출부의 전송 코드를 아래처럼 (기존 엔드포인트 상수가 있다면 URL만 교체):

```javascript
const FORM_ENDPOINT = '할 일 2에서 복사한 /exec URL';

await fetch(FORM_ENDPOINT, {
  method: 'POST',
  headers: { 'Content-Type': 'text/plain;charset=utf-8' },  // ⚠️ 반드시 text/plain
  body: JSON.stringify(payload),  // 설문 응답 객체, email 또는 contact 필드 포함
});
```

⚠️ `application/json`으로 보내면 Apps Script가 preflight(OPTIONS)를 처리하지 못해 **브라우저 CORS에서 차단**돼요. `text/plain`으로 보내야 하고, 수신부(doPost)는 그대로 JSON 파싱해요.

## 할 일 4 — 검증 (순서대로 4개 체크)

1. [ ] 브라우저에서 `/exec` URL 열기 → **"Pickd survey endpoint OK"** 표시
   (안 되면 할 일 2-3 액세스 권한 재확인)
2. [ ] 배포된 `pickd.ai.kr`에서 설문 실제 제출 → DevTools Network에 `exec` 요청 발생
3. [ ] Apps Script 좌측 **실행** 메뉴에 `doPost` "완료됨" 기록
   (기록 자체가 없으면 요청 미도달 = 프론트 URL/캐시 문제 / "실패"면 스크립트 오류 → 오류 메시지 공유)
4. [ ] **시트에 행 추가** 확인 → 테스트 행 삭제

## 운영 주의 (나중에 코드 고칠 때)

- 스크립트 수정 후엔 **배포 관리 > 연필 > 버전 "새 버전" > 배포** 해야 라이브에 반영돼요 (저장만으론 반영 안 됨)
- **"새 배포"를 새로 만들면 URL이 바뀌어요** → 프론트 코드의 URL도 같이 교체 필요. 기존 배포를 "새 버전"으로 수정하면 URL 유지
