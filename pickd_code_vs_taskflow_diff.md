## ✅ 일치하는 부분 (코드 = Task Flow)

### 사이드바 & 내비게이션
- 탭1(공고 대시보드), 탭2(경험정리), 탭3(AI 자소서) 메인 탭 3개
- 캘린더 탭: 사이드바 하단 설정 버튼 위 보조 탭으로 분리 배치
- 설정 탭: 사이드바 최하단

### 탭1 — 공고 대시보드
- 공고 테이블: 필터칩(전체·별표·마감임박·작성중), 별표 토글, 행 클릭별 동작 분기
- 공고 등록 모달: URL / PDF / 이미지 방식 입력 탭 전환
- 상태 관리 모달: 전형 흐름 단계 이동, 연결된 일정·할일 목록
- 인라인 수정: 공고명 외 셀 인라인 편집
- 공고 상세 페이지: 진입 가능
- 대시보드 헤더: 캘린더 바로가기 버튼 (CalendarDays 아이콘, 😊 버튼 왼쪽)

### 탭1 — Today 패널 (홈 우측 슬라이딩 패널)
- 섹션 순서: 미니캘린더 → 마감 임박 공고 → 오늘의 일정 → 오늘의 할일
- 날짜 헤더("오늘 / 6월 25일") 클릭 → 캘린더 탭 이동
- 미니캘린더 월 제목 클릭 → 캘린더 탭 이동
- 오늘의 일정 섹션 제목 클릭 → 캘린더 탭 이동
- 오늘의 할일 섹션 제목 클릭 → 캘린더 탭 이동
- 마감 D-day 색상 구분: D-1 이하 빨강, D-3 이하 주황, 그 외 회색

### 탭2 — 경험정리
- 경험 목록, 기본정보, 파일함 기본 구조
- 상세 에디터 진입

### 탭3 — AI 자소서
- AI 자소서 허브 기본 구조

### 캘린더 탭
- 월간 그리드 + 우측 패널 레이아웃
- 날짜 셀 클릭 → 우측 패널에 해당 날짜 일정·할일·공고 조회
- 우측 패널 내 "전체보기" ↔ "← 날짜별" 버튼으로 패널 내 전환 (별도 오버레이 없음)
- 일정 추가 / 할일 추가 모달 (분리된 2개)
- 공고 필터 (postingFilter)

## ⚠️ 동작이 다른 부분 (코드 ≠ Task Flow)

> 우선 수정 여부를 팀에서 결정해야 하는 항목

### 1. Today 패널 — 마감 임박 공고 항목 개별 클릭
| | 내용 |
|---|---|
| Task Flow | TodayJobClick → **공고 상세 페이지** 이동 |
| 코드 현재 | toast("공고 상세 준비 중") 표시만 |
| 비고 | JobDetail 페이지는 이미 존재(`/job/:id`). 라우팅 연결 필요 |

### 2. Today 패널 — 오늘의 일정 항목 개별 클릭
| | 내용 |
|---|---|
| Task Flow | TodayItemClick → **조회 팝업** (수정·삭제 가능) |
| 코드 현재 | navigate("/calendar")로 바로 이동 |
| 비고 | 조회 팝업(TodayPopup)은 미구현 상태 |

### 3. Today 패널 — 오늘의 할일 항목 개별 클릭
| | 내용 |
|---|---|
| Task Flow | TodayItemClick → **조회 팝업** (수정·삭제 가능) |
| 코드 현재 | done 토글(체크 완료/해제)만 |
| 비고 | 조회 팝업(TodayPopup)은 미구현 상태 |

### 4. Today 패널 — 일정/할일 + 추가 버튼
| | 내용 |
|---|---|
| Task Flow | TodayScheduleAddBtn → **통합 등록폼(RegForm)** 열림. 저장 후 캘린더 탭으로 이동 |
| 코드 현재 | navigate("/calendar")로 이동 (등록폼 없이 캘린더로만 이동) |
| 비고 | 등록폼 통합(RegForm)이 구현되면 자동 해결 |

### 5. 캘린더 — 일정·할일 등록폼
| | 내용 |
|---|---|
| Task Flow | **단일 폼(RegForm)** 내 유형(일정/할일) 자유 전환, 공고 연결 공통 |
| 코드 현재 | CreateScheduleModal + CreateTaskModal **분리된 2개 모달** |
| 비고 | Task Flow에서 후순위 미구현으로 명시됨. 통합 시 ⚠️ 2·3·4번도 함께 해결 가능 |

## 🔶 미구현 — Task Flow에 후순위로 명시된 것

> `pickd_task_flow_v4.md` 상단 및 탭1 섹션에 "미구현(후순위)"으로 이미 표시됨

1. **칸반 뷰** — 공고 테이블 ↔ 칸반 전환 (ViewToggle → Kanban)
2. **완료 공고 CompletedList** — 불합격/포기 후 이동, 파일아이콘/카드뷰 표시
3. **공고 마감일 자동 캘린더 동기화** — 등록 시 AI 추출 마감일 → 캘린더 자동 생성 (AutoCalSync)
4. **Today 패널 조회 팝업** — 항목 클릭 시 팝업으로 수정·삭제 (TodayPopup)
5. **마감일 알림** — Reminder 기능
6. **일정·할일 등록폼 통합** — 단일 RegForm (유형 전환·공고 연결 공통)

## ❌ 미구현 — Task Flow에 있지만 후순위 미표시

> 기획은 확정되어 있으나 코드에 아직 없는 기능. 우선순위 별도 결정 필요.

### 탭1 — 공고 등록·관리
1. **공고 파싱 실패 분기** — AI 파싱 실패 시 재시도 or 수동 입력 전환 선택 (RetryChoice)
2. **중복 공고 감지** — 회사명+직무 동일 시 기존 공고 안내, 기존 보기 or 신규 등록 선택 (DupJobCheck → DupJobAlert)
3. **중복 공고 강제 등록 확정** — 완전 동일 공고를 신규 등록하려는 경우 확정 확인 (DupForceConfirm)
4. **마감일 인라인 수정** — 공고 테이블 마감일 셀 클릭 → DatePicker 인라인 수정 (DeadlinePicker)
5. **연결 일정·할일 호버 메뉴** — 상태관리 모달 내 목록 항목 hover 시 수정·삭제 메뉴 (LinkedItemMenu)
6. **공고 상세 → 자소서 작성하기 버튼** — 탭3(AI 자소서)으로 진입, 공고 자동 선택 (WriteCoverBtn)
7. **공고 상세 → 자소서 반영 내역** — 작성 완료된 자소서 목록 확인 (CoverHistory)
8. **불합격·포기 후 회고 작성 제안** — 최종 결과 입력 후 경험DB 허브(탭2)로 연결 (Retro)

### 탭2 — 경험정리
9. **경험 추가 AI 추출 체크리스트** — 파일 업로드 → AI 추출 완료 후 확인 단계 (Checklist)
10. **중복 경험 감지 및 병합 모달** — 중복 감지 시 병합할 필드 선택 (MergeModal)
11. **경험 목록 부가 기능** — 검색 필터, 카드뷰, 복붙 뷰(핀 고정), 다중 선택, 일괄 삭제, Excel 내보내기

### 탭3 — AI 자소서
12. **SWOT 매칭 → 경험 후보 표시** — 문항 입력 후 AI가 경험DB에서 후보 추출, 사용자 선택
13. **완성된 자소서 → 채용사이트 제출 + 탭1 반영 표시** — SubmitAction → TabBadge

### 캘린더 탭
14. **공고 항목 클릭 → 공고 일정 팝업** — 캘린더 그리드/우측 패널의 공고 클릭 시 팝업 경유, 팝업에서 공고 상세 버튼 (JobSchedulePopup → JobDetailLinkBtn)
15. **일정·할일 수정 진입** — 항목 클릭 → 기존 값이 사전 입력된 등록폼 (현재는 별도 DetailModal)

### 온보딩 (0번 다이어그램)
- 약관 동의, 기본 정보 입력, 관심 채용 유형 선택, 첫 경험 등록, 튜토리얼 전체 플로우 미구현
