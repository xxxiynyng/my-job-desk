### (Phase 1) 병합 UI 검토 결과 (보류 결정)

현재 방식(필드별 라디오 선택)이 스펙 제안 방식(기준경험 편집 + 참고경험 읽기전용 나란히)보다 MVP 단계에서 사용성이 높다고 판단하여 유지.  
**근거**: 필드별 선택 방식은 사용자가 어떤 값을 취할지 명확히 볼 수 있음. 나란히 편집 방식은 좌측 경험을 직접 수정해야 하므로 입력 부담이 높고, 긴 서술형 값에서 혼동 발생 가능성 있음.
